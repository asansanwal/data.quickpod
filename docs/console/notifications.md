# Notifications

Authenticated route: `/notifications`

Login page: [https://console.quickpod.io/log-in](https://console.quickpod.io/log-in)

## What notifications cover

QuickPod notifications provide an account-level feed of operational, billing, and platform events.

## Current notification categories

The console currently exposes category-level preferences for:

* billing
* hosting telemetry
* pods
* storage
* cloud sync
* clusters
* system
* admin

## Current capabilities

The Notifications page lets you:

* read recent notifications
* see unread counts
* mark individual notifications as read
* mark all notifications as read
* mute or unmute categories

## Why this matters

Notifications are now part of the core QuickPod operations experience for both renters and hosts. Older docs that only discussed email or support channels are missing an important part of the modern console.
